Descripción del vídeo Interfaz de Usuario I

Inicio

Este primer vídeo comienza explicando cuáles son las características que debe tener una interfaz para que tenga éxito: que sea atractiva y amigable. Muestra la imagen de un móvil, cuya pantalla tiene una serie de opciones: rutinas, ejercicios, calentamiento y entrenamiento. Cada opción tiene una imagen colorida y descriptiva de la acción.

A continuación menciona las tres formas de cómo crear una interfaz mediante: código en java, lenguaje XML y editando manualmente el componente mediante el panel Attributes. La primera opción no es la más habitual ya que supone muchas líneas de código y tener un alto conocimiento en Java. Expone que muchos de sus lectores en otros cursos le preguntan por qué escribir el código cuando un IDE, por ejemplo Netbeans, puede generar el código por nosotros. En el vídeo el autor expone que es posible que cuando estemos trabajando necesitemos ese conocimiento.

Para demostrar cómo se realiza esta primera opción crea un proyecto en blanco en Android Studio. Abre en el Editor los ficheros MainActivity.java que guarda la parte lógica y content_main.xml que guarda la parte gráfica. Se separa la lógica del diseño por comodidad. El autor quiere modificar el mensaje que aparece por defecto "Hello World". Para ello introduce el siguiente código:

@Override
protected void onCreate(Bundle savedInstanceState) {
   super.onCreate(savedInstanceState);
   //setContentView(R.layout.activity_main);
   TextView textView= new TextView(this);
   textView.setText("Hola alumnos, ¿qué tal estáis?");
   setContentView(textView);
}

Después lo ejecutamos pulsando sobre la opción Run -> Run app y si tenemos el emulador arrancado se verá un mensaje con el texto que se ha añadido. Cuando el autor explica el código se basa en una imagen que muestra cómo todos los componentes, en concreto ViewGroup, TextView e ImageView heredan de la clase View. Para crear una vista hay que tener un elemento que herede de View. 

A continuación el autor recomienda consultar la API de Android que puede consultar en el siguiente enlace Información de la clase View y muestra cómo en la documentación de muestra las clases que heredan de esta clase y los diferentes constructores. Para explicar cómo se crea un componente mediante código hace referencia de nuevo a la documentación oficial de Android Developer y consulta la información de la clase Context que se muestra en el siguiente enlace Información de la clase Context y comprueba como la clase AppCompatActivity  hereda de Context con lo cual el código es correcto. Después comprueba la funcionalidad del método setText() en el siguiente enlace Información de la clase TextView y comprueba como efectivamente con este método se modifica el texto que debe mostrar el componente. Finalmente consulta la documentación de la clase Activity para ver el en qué consiste el método setContentView().
