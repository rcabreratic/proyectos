3.2.1.- Objetivos de la tarea

Objetivos

El objetivo principal de la aplicación será trabajar con la persistencia de datos y las comunicaciones externas. La aplicación será una continuación de la tarea grupal 1, ampliando sus funcionalidades.

Los objetivos específicos de esta tarea están centrados en la utilización de varias tecnologías y enfoques que paso a enumerar:

1. SharedPreferences:

  • Usaremos SharedPreferences para almacenar las opciones de la aplicación, como configuraciones de usuario, preferencias de visualización, temas, etc.
  • Deberás proporcionar una pantalla de preferencias donde los usuarios puedan ajustar estas opciones y luego guarda los valores en SharedPreferences.

2. Ficheros:

  • Se permitirá a los usuarios adjuntar archivos a las tareas y guardar esos archivos en el almacenamiento interno o externo (tarjeta SD) del dispositivo.
  • Vamos a implementar la funcionalidad para adjuntar archivos asociados a cada tarea.
  • Deberemos usar una adecuada gestión de permisos y de ubicación de almacenamiento.

3. Base de datos SQLite interna:

  • Para la persistencia de la colección de tareas, usaremos una base de datos SQLite de Android.
  • Permitiremos a los usuarios consultar las tares almacenadas y también agregar, editar y eliminar tareas, asegurándonos de que estas operaciones quedan reflejadas en la base de datos.

4. Base de datos MySQL/MariaDB externa:

  • Alternativamente a la base de datos interna, deberás crear una base de datos MySQL/MariaDB en un servidor externo que tendrá la misma estructura que la base de datos interna.

5. Librería ROOM

  • La aplicación tendrá un diseño independiente del almacén de datos, es decir, no se verá afectado cuando cambiemos la BD interna por la externa o viceversa. Para esto utilizaremos un repositorio de datos.
  • Deberás usar la librería ROOM para acceder y realizar operaciones de creación, consulta, actualización y borrado del repositorio de datos.

6. Comunicaciones HTTP/HTTPS

  • Se utilizarán clases que permitan comunicar la aplicación, mediante HTTP/HTTPS, con la base de datos externa.
