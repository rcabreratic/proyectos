Apartado 1. Shared Preferences (3 puntos)

La actividad "Preferencias" será accesible desde el menú de la aplicación (no como un botón de acción). Para acceder a ella, añadid una nueva entrada que aparecerá en la primera posición del menú, encima de "Acerca de". La actividad deberá mostrar una barra de acción con el título "Preferencias" y con el botón "Home" que servirá para salir de la actividad y volver al listado. Además deberá incluir las siguientes preferencias agrupadas por categorías y presentarlas con iconos.

1. CATEGORÍA "VISUALIZACIÓN"

  • Tema (Switch con textos "claro" y "oscuro", Preferencia "tema"):
    • Tipo: Valor booleano (depende de "contraste"). Valor por defecto true.
    • Descripción: True indica un tema claro para la aplicación; false indica un tema oscuro. Este ajuste transformará completamente el aspecto de la aplicación. (Nota: Podéis utilizar los recursos res/values/styles.xml para el tema claro y res/values-night/styles.xml para el tema oscuro).
  • Alto Contraste (Checkbox. Preferencia "contraste"):
    • Tipo: Valor booleano. Valor por defecto false.
    • Descripción: True indica un tema de alto contraste que ajustará el tema de la aplicación para mejorar la visualización de personas con daltonismo. Si esta preferencia está en true, deshabilitará la preferencia "tema"; si está en false, habilitará la preferencia "tema", estableciendo así un tema claro u oscuro.
  • Tamaño de Letra (Lista de valores. Preferencia "fuente"):
    • Tipo: Valores mostrados: pequeña, mediana y grande. Valores leídos: 1, 2 o 3. Valor por defecto mediana (2).
    • Descripción: Deberá mostrarse en la preferencia el valor seleccionado. Este ajuste modificará el tamaño de la letra en toda la aplicación. (Nota: Puede ser útil utilizar un archivo de dimensiones /res/values/dimens.xml).

2. CATEGORÍA "ORDENACIÓN"

  • Criterio (Lista de valores. Preferencia "criterio"):
    • Tipo: Valores mostrados: alfabético, fecha de creación, días restantes, progreso. Valores leídos: 1, 2, 3 o 4. Valor por defecto fecha creación (2).
    • Descripción: Deberá mostrarse en la preferencia el valor seleccionado. Cambiará el orden de presentación de las tareas según el criterio elegido: alfabético, por fecha de creación, por días restantes o por el estado de progreso. La preferencia "criterio" influirá en esta ordenación.
  • Orden (Switch con textos "asc" y "desc". Preferencia "orden"):
    • Tipo: Valor booleano. Valor por defecto true.
    • Descripción: True indica orden ascendente; false indica orden descendente. Este criterio se aplicará a cualquiera de los criterios de ordenación mencionados anteriormente.

3. CATEGORÍA "ALMACENAMIENTO"

  • Almacenamiento en tarjeta SD (Checkbox. Preferencia "sd"):
    • Tipo: Valor booleano. Valor por defecto false.
    • Descripción: True indica que los archivos adjuntos se guardarán en la tarjeta SD si está presente; false indica que se guardarán en la memoria interna del dispositivo.
  • Limpieza de archivos (Número. Preferencia "limpieza"):
    • Tipo: Valor entero. Valor por defecto 0.
    • Descripción: Este valor, expresado en días, determina la periodicidad con la que la aplicación llevará a cabo la eliminación automática de archivos considerados antiguos. Un valor establecido de, por ejemplo, 30 días, implicará que la aplicación eliminará automáticamente todos los archivos adjuntos que no hayan sido abiertos en dicho periodo, es decir, aquellos que se consideran antiguos. El valor 0 se interpreta como una duración indefinida, indicando que no se realizará limpieza automática. En este caso, los archivos adjuntos permanecerán en el sistema hasta que el usuario decida eliminarlos manualmente o se elimine la tarea.
  • Base de Datos (Switch con textos "ext" e "int". Preferencia "bd"):
    • Tipo: Valor booleano. Valor por defecto false (interna).
    • Descripción: True indica conexión a una base de datos externa MySQL o MariaDB; false indica conexión a una base de datos local SQLite. Esta preferencia es clave para las configuraciones siguientes. Si es false, todas las preferencias a continuación deben estar deshabilitadas; si es true, deben estar visibles y editables.

3.1 SUBCATEGORÍA "BASE DE DATOS EXTERNA"

  • Nombre Base de Datos (Texto. Preferencia "nombrebd"):
    • Tipo: Valor textual (depende de "bd"). Valor por defecto "bd".
    • Descripción: Recoge el nombre de la base de datos externa.
  • IP (Texto. Preferencia "ip"):
    • Tipo: Valor textual (depende de "bd"). Valor por defecto "10.0.2.2".
    • Descripción: Almacena la dirección IP o URL del servidor de base de datos. (Nota: La IP 10.0.2.2 permite realizar pruebas contra bases de datos locales desde el dispositivo virtual)
  • Puerto (Texto. Preferencia "puerto"):
    • Tipo: Valor textual (depende de "bd"). Valor por defecto "1001".
    • Descripción: Recoge el número de puerto.
  • Usuario (Texto. Preferencia "usuario"):
    • Tipo: Valor textual (depende de "bd"). Valor por defecto "usuario"
    • Descripción: Almacena el nombre de usuario de la base de datos.
  • Contraseña (Texto. Preferencia "password"):
    • Tipo: Valor textual oculto (depende de "bd"). Valor por defecto "" (cadena vacía).
    • Descripción: Almacena la contraseña del usuario.

4. Botón de "Restablecer":

  • Tipo: Se trata de una dependencia personalizada que solo se va a usar para colocar un layout con un botón.
  • Descripción: Aparecerá un botón para restablecer las preferencias de usuario a los valores predeterminados. (Nota: Será de utilidad el atributo xml onClick para asignar un método escuchador al botón que tendrá que estar implementado en la actividad "Preferencias")
