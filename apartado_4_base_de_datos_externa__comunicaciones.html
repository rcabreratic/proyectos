Apartado 4: Base de datos externa + comunicaciones

Implementad ahora la misma base de datos utilizando un servidor externo. Podéis hacer las pruebas en local usando un servidor MySQL/MariaDB. (Nota: Os propongo que indaguéis sobre los contenedores Docker para levantar el servicio de Base de Datos de forma limpia y con el mínimo esfuerzo)

  • Para utilizar un servicio externo podréis utilizar la librería de Android HttpURLConnection, aunque también se pueden utilizar otras librerías externas más avanzadas como Volley, OkHttp o Retrofit. Estas librerías os permitirán manejar comunicaciones mediante los protocolos HTTP/HTTPS. El servicio lo proporcionará una API externa que haga consultas a la Base de Datos.
  • De forma alternativa se podrá utilizar un driver JDBC para hacer las consultas directamente desde la aplicación cliente. El grupo debe sopesar la opción más adecuada.
  • Maneja las operaciones de manera asíncrona para no bloquear el hilo principal de la interfaz de usuario. En su caso, deberás implementar hilos mediante las clases Thread o ExecutorService. Para acceder al hilo principal, donde se ejecuta la interfaz de usuario, utiliza la clase Handler.
  • Por último, debes tener en cuenta la correcta sincronización entre las bases de datos local y externa para garantizar la coherencia de la información.
