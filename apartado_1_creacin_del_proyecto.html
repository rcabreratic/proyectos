Apartado 1. Creación del proyecto

Instrucciones:

  1. Creación de un nuevo proyecto en Android Studio y agrega las clases necesarias:
    1. MainActivity: La actividad inicial de la aplicación.
    2. ListadoTareasActivity: La actividad que contendrá la lista de tareas.
    3. CrearTareaActivity: La actividad que servirá para crear una tarea.
    4. EditarTareaActivity: La actividad que servirá editar una tarea.
    5. Tarea: La clase que representa una tarea. Debe contener atributos para: título, descripción, progreso (valor entero entre 0 y 100), fecha creación, fecha objetivo, prioritaria (sí/no).
    6. TareaAdapter: El adaptador que se utiliza para mostrar las tareas en la lista.
    7. Cualquier otra clase que sea necesaria para el funcionamiento de la aplicación.
  2. Desarrollo de la interfaz gráfica:
    1. La interfaz gráfica debe permitir a los usuarios listar, agregar, editar y eliminar tareas.
    2. Utilice las clases gráficas adecuadas para cada elemento de la interfaz gráfica.
    3. Personaliza mediante temas y estilos la aplicación. Debes utilizar algún tema para la aplicación con una gama de colores coherente y tipografías adecuadas. Además personalizarás los estilos de botones, vistas de texto, etc.
  3. Realización de pruebas de interacción:
    1. Probamos la aplicación utilizando un emulador de Android y un dispositivo físico.
    2. Nos aseguramos de que la aplicación funciona y se muestra correctamente para realizar todas las operaciones en disposición vertical y en horizontal (creamos layouts específicos para cada orientación).
    3. Probamos a cambiar el idioma del dispositivo (a inglés) y comprobamos que la aplicación queda completamente traducida.
  4. Empaquetado y despliegue de la aplicación:
    1. Creamos un icono para la aplicación.
    2. Empaquetamos la aplicación en un archivo .apk.
    3. Desplegamos la aplicación en un dispositivo móvil Android o en un emulador de Android.
  5. Documentación del proceso de desarrollo:
    1. Reunimos documentación sobre las clases utilizadas, las decisiones de diseño, las dificultades encontradas y cómo se resolvieron, así como cualquier otra información relevante.
