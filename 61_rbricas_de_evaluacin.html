2.a) Generación de estructura de clases

Estructura ausente o incorrecta. No se han definido las clases necesarias.

Estructura básica definida, pero con errores o inconsistencias.

Estructura de clases adecuada, pero con algunas deficiencias.

Estructura de clases bien definida y coherente.

2.b) Utilización de clases para aplicaciones gráficas

No se han utilizado clases para la interfaz gráfica.

Utilización mínima de clases para la interfaz gráfica.

Utilización adecuada de clases para la interfaz gráfica, pero con algunas deficiencias.

Utilización eficaz y apropiada de clases para la interfaz gráfica.

2.g) Pruebas de interacción usuario-aplicación

No se han realizado pruebas de interacción.

Pruebas mínimas de interacción.

Pruebas de interacción adecuadas, pero con algunas deficiencias.

Pruebas exhaustivas de interacción y optimización de la aplicación.

2.i) Documentación del proceso de desarrollo

Documentación ausente o incompleta.

Documentación básica, pero con carencias significativas.

Documentación adecuada, pero con algunas deficiencias.

Documentación completa y detallada del proceso de desarrollo.

3.b) Reconocimiento de clases multimedia

No se han identificado las clases necesarias para la captura y procesamiento multimedia.

Reconocimiento mínimo de algunas clases multimedia, pero incompleto.

Reconocimiento adecuado de clases multimedia, pero con algunas deficiencias.

Identificación clara y completa de todas las clases multimedia relevantes.

3.c) Utilización de clases para la conversión de datos multimedia

No se han utilizado clases para la conversión de datos multimedia.

Utilización mínima de clases para la conversión de datos multimedia.

Utilización adecuada de clases para la conversión de datos multimedia, pero con algunas deficiencias.

Utilización eficaz y apropiada de clases para la conversión de datos multimedia.

3.h) Depuración y documentación de programas

Depuración y documentación ausentes o incompletas.

Depuración y documentación básicas, pero con carencias significativas.

Depuración y documentación adecuadas, pero con algunas deficiencias.

Depuración completa y documentación detallada de los programas desarrollados.
