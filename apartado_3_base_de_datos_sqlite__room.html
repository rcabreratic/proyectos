Apartado 3: Base de datos SQLite + ROOM

Las informaciones a almacenar en la base de datos con respecto a cada noticia se detallan a continuación:

  • Id: Campo numérico autoincrementable, que actuará como clave principal.
  • Título: Campo de texto destinado a contener el título de la noticia. Campo no nulo.
  • Fecha de creación: Campo de fecha designado para almacenar la fecha en que se creó la noticia. Campo no nulo. Valor por defecto, hoy.
  • Fecha objetivo: Campo de fecha designado para almacenar la fecha de vencimiento de la noticia. Campo no nulo. Valor por defecto, la fecha de hoy más una semana.
  • Progreso: Campo numérico (rango: 0 a 100) que indica el porcentaje de completitud de la noticia. Valor por defecto, 0.
  • Prioritaria: Campo booleano que registra si la noticia se considera prioritaria o no. Valor por defecto, falso.
  • Descripción: Campo de texto destinado a contener la descripción detallada de la noticia. Campo opcional.
  • URL_doc: Campo de texto que indica la ubicación de un archivo de documento asociado a la noticia. Campo opcional
  • URL_img: Campo de texto que especifica la ubicación de un archivo de imagen vinculado a la noticia. Campo opcional.
  • URL_aud: Campo de texto que indica la ubicación de un archivo de audio asociado a la noticia. Campo opcional
  • URL_vid: Campo de texto que especifica la ubicación de un archivo de vídeo vinculado a la noticia. Campo opcional.

Consultas o filtrados que se pueden hacer:

  1. Consulta de todas las tareas o solamente las tareas prioritarias. Tendréis que tener en cuenta todas las posibilidades contempladas en las preferencias del usuario en cuanto al criterio de ordenación y el sentido ascendente o descendente.
  2. Añadid una entrada en el menú de la aplicación que aparecerá en tercera posición, entre las opciones "Acerca de" y "Salir", llamada "Estadísticas". Deberéis mostrar informaciones de estadísticas sobre las tareas. Diseñad esta sección con libertad, podéis mostrar informaciones como el promedio de progreso, la cantidad de tareas en cada estado, la fecha objetivo promedio .... Esto podría ser útil para proporcionar a los usuarios información sobre su progreso general.

(Nota: Deberéis utilizar la librería ROOM con un repositorio de datos ya que es una buena práctica para abstraer la capa de persistencia de datos del resto de la aplicación, proporcionando una interfaz limpia y fácil de usar para interactuar con la base de datos. Al aislar la lógica de la base de datos en un repositorio, se logra una separación clara de responsabilidades: el repositorio de datos actúa como un intermediario entre la capa de datos y las demás partes de la aplicación, mejorando la coherencia y la modularidad del código).
