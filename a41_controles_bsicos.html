A.4.1.- Controles básicos

Llegados a este punto, nos planteamos algunas cuestiones,  como las siguientes:

CC by Mårten Björk
  • ¿Cómo puedes utilizar las aplicaciones instaladas en tu dispositivo móvil?
  • ¿Cómo rellenas un formulario con tus datos personales?
  • ¿Utilizas algún botón para confirmar alguna acción como “enviar”?

Pues bien, todas estas acciones las podemos llevar a cabo porque las diferentes pantallas por las que navegamos en nuestra aplicación disponen de elementos o controles predeterminados que nos permiten interactuar con el dispositivo. Veremos a continuación cuáles son.

TexView

Muestra un campo de texto al usuario. Este texto no es editable por el usuario. Se podrá modificar su estilo mediante los siguientes atributos:

  • android:text: mediante este atributo se establece el texto por defecto. Se utilizará el recurso @string para poder mostrar el texto en diferentes idiomas. 
  • android:background: permite establecer el color de fondo del Textview.
  • android:textColor: es un valor en hexadecimal que indica el color del texto.
  • android:textSize: en un valor en sp que indica el tamaño del texto. 
  • android:textStyle: permite establecer el estilo de texto en normal, negrita (bold) o cursiva (italic).

EditText

Es un campo de texto en el que, a diferencia de TextView, el usuario puede editar su contenido para introducir un dato en la aplicación. Se pueden utilizar los mismos atributos que para el control TextView y además los siguientes:

  • android:inputType: este atributo es importante porque indica el tipo de texto que el usuario introducirá. En base al tipo de entrada el sistema operativo configura el tipo de teclado que se muestra, los caracteres aceptables y la apariencia del texto de edición. Con android:inputType="phone" indicamos que el usuario sólo puede introducir caracteres válidos en un número de teléfono. Puedes comprobar cómo se visualiza el teclado en la documentación oficial: Cómo especificar el tipo de método de entrada.

    Si se quiere poner más de un valor a un atributo se puede utilizar el carácter pipe "|" para separar los valores. 

  • android:hint: sirve para introducir un texto semitransparente que desaparece cuando el usuario pulsa sobre el EditText.
  • android:textColorHighlight: es un valor en hexadecimal que indica el color del texto seleccionado.

Button

Es el control básico que el usuario pulsa para realizar una acción. Esta acción nosotros la programamos en el evento onClick(View v). Se puede utilizar el atributo android:onclick="sendMessage" de forma que se referencie un método definido en la clase Activity que tenga la siguiente declaración: 

public void sendMessage(View view)

Es el sistema operativo quien ejecuta el método sentMessage automáticamente cuando el usuario pulsa el botón sin tener que implementar el escuchador onClickListener en el botón.

Switch

Es un botón a modo de interruptor que  tiene dos estados posibles: encendido y apagado. Se pueden usar los siguientes atributos:

  • android:checked: para modificar su estado.
  • android:text: para establecer el texto en base a su estado 

CheckBox

Se utiliza para mostrar un listado de opciones que el usuario puede marcar o desmarcar. Este componente hereda indirectamente de TextView con lo cual se le pueden asignar los atributos que se han explicado además de los que listan a continuación:

  • android:checked para inicializar el estado del control. Se asignará true si queremos que esté marcado o en otro caso false.

RadioButton

Al igual que CheckBox puede tener dos valores: marcado o desmarcado con la diferencia que las opciones forman un grupo mediante la clase RadioGroup y sólo puede haber una opción marcada. Es decir,  si se marca una de las opciones se desmarcará automáticamente la que estuviera activa anteriormente.

ImageButton

Tiene la misma funcionalidad que Button pero se muestra una imagen en vez de texto. Se deben utilizar los siguientes atributos:

  • android:src para indicar el recurso imagen que se encuentra en /res/drawable. Por ejemplo para acceder a una imagen ic_calendar sería @drawable/ic_calendar.
  • android:contentDescription: sirve para describir mediante texto la imagen del botón de forma que nuestra aplicación sea lo más accesible posible.

ImageView

Este control sirve para introducir una imagen en nuestra aplicación a modo decorativo porque no tiene funcionalidad. Se utiliza el atributo android:src para indicar la imagen a visualizar.

FloatingActionButton

Este componente aparece con Material Design y se trata de un botón redondo que se eleva por encima del contenido de la pantalla principal y permite al usuario realizar un acción que se destaca entre las demás. Por ejemplo, en un listado suele ser la opción Añadir o Add un elemento a la lista. Los botones de acción flotante vienen en un tamaño predeterminado y con unos atributos comunes donde se suele modificar el color que lo rellena mediante el atributo colorAccenty el icono que lo identifica con el atributo android:src.

<android.support.design.widget.FloatingActionButton
android:id="@+id/fab"
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:layout_gravity="end|bottom"
android:src="@drawable/ic_action_add_icon"
android:contentDescription="@string/action_add"
android:layout_margin="16dp" />

Hasta ahora hemos visto los diferentes tipos de controles más utilizados, pero si te fijas en la paleta que incorpora Android Studio puedes insertar en tus aplicaciones muchos controles más como por ejemplo: TextClock, CalendarView, ProgressBar, RatingBar, etc. Puedes probarlos en tus aplicaciones y consultar toda la información de cada una de las clases en la documentación oficial de Android.
